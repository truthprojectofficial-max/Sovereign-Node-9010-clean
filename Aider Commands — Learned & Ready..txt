Detailed Breakdown of Top Methods
1. Automated Multi-Turn Red Teaming (Current Best Practice)

One model (the "attacker") is given the goal of making the target model violate its rules.
It runs for dozens or hundreds of turns, learning what works.
Tools: garak, Promptfoo, custom frameworks from Anthropic/DeepMind/OpenAI.
Extremely effective at finding subtle, context-dependent failures.

2. LLM-as-Attacker / Adversarial Prompt Generation

Use a strong model (often the same family or a more capable one) to generate thousands of potential jailbreaks.
Then filter and test the best ones.
Many open-source tools now exist for this (e.g., PAIR, TAP, AutoDAN).

3. Human Expert Red Teaming

Still considered the gold standard for discovering novel attack vectors.
Used heavily by labs before release (Anthropic, OpenAI, Google).
Very expensive and slow, but finds things automation misses.

4. Persistent Red Teaming

Attacks that last over many sessions or use memory.
Particularly dangerous because models can be "groomed" over time.
